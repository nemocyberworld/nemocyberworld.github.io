Sock Puppets & Operational Security (OPSEC)

OPSEC Tools & Techniques

Tools

Tails OS · Qubes OS · Tor · Mullvad VPN · ProtonVPN · VeraCrypt · KeePassXC

Techniques

  • Run investigations inside isolated VMs or live OS (Tails/Qubes)
  • Create one sock-puppet persona per purpose (email, social, notes)
  • Use VPN → Tor chaining for stronger anonymity
  • Strip metadata before uploads (ExifTool)
  • Encrypt notes & evidence (VeraCrypt, zip AES)
  • Hash originals for provenance (sha256sum file)
  • Rotate activity times to avoid fingerprinting by behavior
  • Snapshot VMs and roll back to clean state after ops

sha256sum photo.jpg > photo.sha256

Obsidian, Graph Notebooks & Offline Graphs

Triage Tools & Techniques

Tools

Obsidian · Dataview Plugin · Obsidian Git · Logseq · Neo4j · Gephi · GraphXR · Kumu · yEd Live

Techniques

  • Use **Obsidian/Logseq** vaults as canonical evidence notebooks.
  • Standardize YAML frontmatter: IDs, aliases, sources, confidence.
  • Link entities with `related:` fields for graph exports.
  • Export nodes/edges as CSV → load into Neo4j, Gephi, or GraphXR.
  • Automate vault-to-graph sync with scripts or Obsidian Git.
---
id: org-001
type: Organization
name: "Acme Corp"
aliases: ["acme-inc"]
source_urls: ["https://opencorporates.com/companies/gb/01234567"]
confidence: high
related:
  - id: person-123
    relation: director
---
table id, type, name, confidence
from "entities"
where type
sort created desc
"id","label","type","name","confidence"
"org-001","Organization","Org","Acme Corp","high"
"source","target","relation"
"person-123","org-001","director"
LOAD CSV WITH HEADERS FROM 'file:///nodes.csv' AS r
MERGE (n:Entity {id: r.id})
SET n.type = r.type, n.name = r.name;

LOAD CSV WITH HEADERS FROM 'file:///edges.csv' AS r
MATCH (a:Entity {id: r.source}), (b:Entity {id: r.target})
MERGE (a)-[:REL {type:r.relation}]->(b);

Pro tips (Expert)

  • Use **persistent IDs** (`org-001`, `person-123`) to avoid duplicates.
  • Keep **provenance fields** (`captured_at`, `evidence_id`) in notes.
  • Automate **JSON → CSV → Graph** pipelines with Python.
  • Maintain **offline vaults** and encrypted backups for sensitive cases.
  • Prefer lightweight tools (yEd, Kumu) for fast triage before deep Neo4j work.

Final: OPSEC Golden Rules & Reporting

Policy Tools & Techniques

Tools

Hunchly · Archive.today · Internet Archive · Zotero · EFF Privacy Tools · ICRC OSINT Guidelines · GPG / OpenPGP

Techniques

  • Separation: run investigations in VMs, Tails/Qubes, or dedicated hardware.
  • Compartmentalization: use burner accounts, disposable emails, isolated VPNs.
  • Provenance-first: capture raw HTML, JSON, screenshots, and hash with sha256sum.
  • Encrypt & backup: archive cases with AES/GPG and keep redundant secure copies.
  • Report integrity: timestamp and checksum evidence; keep manifests for reproducibility.

Example Commands

zip -er case_CFR-2025-001_raw.zip captured_files/ evidence_manifest.csv

Pro Tips

  • Keep one canonical encrypted bundle per case with an audit log.
  • Enforce access controls & retention schedules for evidence.
  • Document queries, API calls & timestamps for reproducibility.
  • Separate executive summary (findings & risks) from technical appendix (evidence, manifests).
  • Use timestamping services to prove evidence integrity.
